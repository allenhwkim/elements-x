Install and Setup

$ nvm ls-remote
$ nvm install --lts
$ nvm use --lts

Install A nuxt starter package

$ nvm use --lts
$ npm install -g vue-cli
$ vue init nuxt/starter portfolio
$ cd portfolio
$ npm install
$ npm run dev $ or, nuxt

`{ rel: 'stylesheet', href: '`[https://unpkg.com/bulma@0.6.2/css/bulma.css`](https://unpkg.com/bulma@0.6.2/css/bulma.css)' }`

In addition to the existing structure, you are going to add or update the following.

  • navigation bar, components/navbar.vue

  • layout page, layouts/default.vue

  • home page, pages/index.vue

  • about page, pages/about.vue

  • projects page, pages/projects.vue

  • contact page, pages/contact.vue

Where is router defined?

Nuxt.js automatically generates the vue-router configuration based on your file tree of Vue files inside the pagesdirectory.

Build For Production

$ nuxt generate
$ npm install -g http-server
$ nuxt generate # to generate `dist` directory
$ cd dist
$ http-server # to see how it works

Your our dist directory is to be used in your production environment.

You can find all code used here at https://github.com/allenhwkim/vue-nuxt-practice/tree/1.0

As a demo, this application has been deployed to https://vue-nuxt-practice.firebaseapp.com/. Do you want to deploy your front-end application to Firebase? Then, you need to create an account for firebase and create a project then run the following.

$ npm install -g firebase-tools --save-dev
$ firebase login --reauth
$ firebase init
$ firebase deploy

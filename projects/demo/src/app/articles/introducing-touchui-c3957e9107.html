Why not fire touch events?

Instead of callbacks or promise?

TouchUI is a open-source library that fires touch event on a registered element. It doesn’t have any dependencies, and it’s very small, only 4.6 kB minified + gzipped!

    • Usage

    It’s easy to use. For default behaviour, just include the library and create an instance.

    new TouchUI();

    By default, all elements of HTML document is ready to fire touch events; tab, double-tab, triple-tab, hold, tap-and-hold, and double-tap-and-hold . All you need to do is add an event listener to the element.

    var myEl = document.getElementById('my-touch');
myEl.addEventListener('tap', function(e) {
  console.log(e);
}

    There are many events that are supported from this library.

    new TouchUI() makes any element of document to fire the following events;

  • tap / double-tap / triple-tap
  • hold / tap-and-hold / double-tap-and-hold

    • TouchUI.draggable(el) makes the element to fire the following events including allnew TouchUI() events.

  • drag-start / drag-move / drag-end

    • TouchUI.droppable(el) makes the element to fire the following events including allnew TouchUI() events.

  • drag-enter / drag-leave / drop

    • The same applies to TouchUI.swipable(el).

  • swipe-left / swipe-right / swipe-up / swipe-down

    • The same applies to TouchUI.pannable(el).

  • pan-start / pan-move / pan-end

    • The same applies to TouchUI.resizable(el).

  • resize-start / resize-move / resize-end

    • There are some recommendation when you want your HTML touchable. This is not only for this library but also for all touch-desired web pages.

  • meta tag is recommended, it gives more control back to the webpage by disabling unnecessary zooming.
    • <head>
  <meta name="viewport" content="user-scalable=no, width=device-width, initial-scale=1, maximum-scale=1">
</head>

    Notes

    Why another touch Javascript instead of others? The main reason is to have Javascript event fired for touch events, not callbacks. Also, it is very focused on simplicity and design. In the result,

  • All events fired by this library work for desktop browsers, of course, as well as mobile browsers.
  • It does not deal with multi-touch events. e.g. two finger zoom, two finger swipe, etc. The reason is that each mobile browser has its own way of dealing with multi-touch events and this library does not make another hacks on the top of the default behaviour, which makes less maintainable and more complicated.
  • It fires a Javascript event without using any callbacks, so that you can add event listener like a vanilla Javascript. This is unlike other touch libraries, which you have to learn their own way to use their own callbacks.

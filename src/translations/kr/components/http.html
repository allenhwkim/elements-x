HTTP

HTTP 콜(GET / POST / PUT / DELETE / PATCH)을 실행합니다. 그리고 콜이 실행뒬때 로딩사인을 보여줍니다.

속성

  • url: 필수, 예 https://my.api.com/v1/users?q=foo.
  • method: 기본값. "get". Method of http call, "get", "post", "put", "delete", "patch".
  • auto: 기본값 false. auto일 경우 초기화시 실행되거나 payload가 변경될경우 실행됩니다.
  • resptype: 기본값 "json". response 타입. 이것은 response의 parser를 결정합니다.
  • resp-status-code: 기본값 undefined. 이 값이 주어졌을 경우 status code를 HTTP의 response에서 찾는게 아니라 주어진 response의 표현에서 찾습니다. 예 "status.code"
  • headers: 기본값 undefied. 이 값이 주어질 경우, HTTP콜을 하기전에 Header를 주어진 값에 따라 설정합니다.
    예, headers="Authorization: Bearer r3f3f2d63f2f23"
  • payload: 기본값 undefined. 이 값이 주어질 령우, HTTP콜을 하기전에 payload를 주어진 값에 따라 설정합니다.
    예, payload="gender=Male;status=Active

이벤트

  • x-success: HTTP콜이 성공적으로 이루어질 경우, 즉 status code가 200대일 경우, 에 발생하는 이벤트.
  • x-errorr: HTTP콜에서 에러가 나는 경우 발생하는 이벤트

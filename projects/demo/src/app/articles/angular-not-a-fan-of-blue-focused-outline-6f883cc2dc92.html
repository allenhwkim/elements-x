Angular: Not a Fan of Blue Focused Outline?

However, it‚Äôs necessary for accessibility

As webaim.org says

The outline CSS property allows you to define the visual outline that displays when elements are selected or have focus. Otherwise known as the focus indicator, it is seen as the dotted lines that surround links and buttons when you tab to them with the keyboard.

To see the benefit of outline, you can press the tab key multiple times to see which element is focused at the moment. Imagine yourself being not able to use mouse, blind or injured. It is a very important feature for them because this feature will tell you what elements are currently focused on.

However,

You may think that it is not a necessary feature to highlight the focused element because you can see clearly and you use mouse every day. You may feel like this feature like a stair ramp or different language announcement in an airplane.

Satisfy All

How? For mouse users, hide outline, and for keyboard users, show outline.

The following is the steps how you implement this in Angular

  • Create a component, a11y-outline.component.ts
  • Apply the component to your app.

    • Create a component, a11y-outline.component.ts

    import { Component, HostBinding, HostListener } from '@angular/core';
    @Component({
  selector: '[a11y-outline]',
  template: `<ng-content></ng-content>`,
  styles: [`
    :host:not(.a11y-outline) ::ng-deep :focus {
      outline: none !important;
    }
  `]
})
export class A11yOutlineComponent  {
  @HostBinding('class.a11y-outline') outline;
  @HostListener('document:click') docClick() {this.outline = 0;}
  @HostListener('document:keydown.tab') docTab() {this.outline = 1;}
}

    Apply it to your app

    Declare it to app.module.ts

    import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { FormsModule } from '@angular/forms';
import { AppComponent } from './app.component';
import { A11yOutlineComponent } from './a11y-outline.component';
    @NgModule({
  imports:      [ BrowserModule, FormsModule ],
  declarations: [ AppComponent, A11yOutlineComponent ],
  bootstrap:    [ AppComponent ]
})
export class AppModule { }

    app.component.html

    <div class="main" a11y-outline>
  <p>
    Use keyboard tab or mouse to navigate.
  </p>
  <button>Button</button>
  <button>Button</button>
  <button>Button</button>
  <br/>
  <input placeholder="Focus here"/>
  <input placeholder="Focus here"/>
</div>

    That‚Äôs It.

    To explain how it works

    It adds a11y-outline class to your app component when tab key is pressed, and it removes a11y-outline from your app component. a11y is a shortcut of ‚Äúaccessibility‚Äù, 11 characters.

    @HostBinding('class.a11y-outline') outline;
@HostListener('document:click') docClick() {this.outline = 0;}
@HostListener('document:keydown.tab') docTab() {this.outline = 1;}

    When class a11y-outline is not set on your application, any focused element will not have outline.

    styles: [`
    :host:not(.a11y-outline) ::ng-deep :focus {
      outline: none !important;
    }
  `]
})

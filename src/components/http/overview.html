HTTP

It makes http call: GET, POST, PUT, DELETE, or PATCH. It also shows loading sign when a call is made.

Attributes

  • url: Required, e.g. https://my.api.com/v1/users?q=foo.
  • method: Default. "get". Method of http call, "get", "post", "put", "delete", or "patch".
  • auto: Default false. When it's auto, it makes http call when initialized or its payload changes.
  • resptype: Default "json". Indicates response type, it determines the parser of response.
  • resp-status-code: Default undefined. When it's given, it determines status code from the given expression. e.g. "code"
  • headers: Default undefied. When it's given, the header is set for each call.
    e.g., headers="Authorization: Bearer r3f3f2d63f2f23"
  • payload: Default undefined. When it's given, it is used to build body of http call.
    e.g., payload="gender=Male;status=Active

Events

  • x-http-success: This event is fired when http call is successful, status code is less than 300.
  • x-http-errorr: This event is fired when http call is NOT successful

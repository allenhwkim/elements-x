Webpack For Dummies

Basic Concept of Webpack; entry, output, loaders, and plugins

What is webpack? webpack is a package bundler for your Javascript application. This is to give basic idea of how to create your own webpack.config.js.

There are four terms to know, and itâ€™s all about input and output.

Concepts

What is your input? entry
What is your output? output
What to do with input? loaders
What to do with output? plugins

Simple Example

index.js

import bar from './bar';
bar();

bar.js

export default function bar() {
  //
}

webpack.config.js

module.exports = {
  entry: './index.js',
  output: {
    filename: 'bundle.js'
  }
};

index.html

<html>
  <head> ... </head>
  <body>
    ...
    <script src="bundle.js"></script> // this has index.js,
  </body>
</html>

Getting Started

$ npm install --save-dev webpack 
## create a webpack.config.js
$ webpack

webpack.config.js

entry

module.exports = {
  entry: './path/to/my/entry/file.js'
};

output

module.exports = {
  ...
  output: {
    filename: 'my-first-webpack.bundle.js'
  }
};

loaders

module.exports = {
  ...
  module: {
    rules: [
      { test: /\.ts$/, use: 'ts-loader'}
    ]
  }
}

plugins

module.exports = {
  ...
  plugins: [
    new webpack.optimize.UglifyJsPlugin()
  ]
}

webpack for development

$ npm install webpack-dev-server --save-dev
$ node_modules/.bin/webpack-dev-server
